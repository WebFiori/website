Class Functions Details

public function function &getActionByName(string $actionName)
Returns an API action given its name.

Parameters:

  • string $actionName The name of the action.
public function function __construct(string $version)
Creates new instance of the class. By default, the API will have two actions added to it:
  • api-info
  • request-info
The first action is used to return a JSON string which contains all needed information by the front-end to implement the API. The user can supply an optional parameter with it which is called 'version' in order to get information about specific API version. The second action is used to get basic info about the request.

Parameters:

  • string $version [Optional] initial API version. Default is '1.0.0'
public function function actionNotImpl()
Sends a response message to indicate that an action is not implemented. This function will send back a JSON string in the following format:

{
  "message":"Action not implemented.",
  "type":"error",
}

In addition to the message, The response will sent HTTP code 404 - Not Found.
public function function actionNotSupported()
Sends a response message to indicate that an action is not supported by the API. This function will send back a JSON string in the following format:

{
  "message":"Action not supported",
  "type":"error"
}

In addition to the message, The response will sent HTTP code 404 - Not Found.
public function function addAction(APIAction $action, boolean $reqPermissions)
Adds new action to the set of API actions.

Parameters:

  • APIAction $action The action that will be added.
  • boolean $reqPermissions [Optional] Set to TRUE if the action require user login or any additional permissions.
public function function contentTypeNotSupported(unkown_type $cType)
Sends a response message to indicate that request content type is not supported by the API. This function will send back a JSON string in the following format:

{
  "message":"Content type not supported.",
  "type":"error",
  "request-content-type":"content_type"
}

In addition to the message, The response will sent HTTP code 404 - Not Found.

Parameters:

  • unkown_type $cType [Optional]
public function function databaseErr(JsonI|JsonX|string $info)
Sends a response message to indicate that a database error has occur. This function will send back a JSON string in the following format:

{
  "message":"Database Error",
  "type":"error",
  "err-info":OTHER_DATA
}

In here, 'OTHER_DATA' can be a basic string or JSON string. Also, The response will sent HTTP code 404 - Not Found.

Parameters:

  • JsonI|JsonX|string $info [Optional] An object of type JsonI or JsonX that describe the error in more details. Also it can be a simple string or JSON string.
public function function getAction()
Returns the action that was requested to perform.
public function function getDescription()
Returns the description of the API.
public function function getInputs()
Returns an array of filtered request inputs.
public function function getInvalidParameters()
Returns an array that contains the names of API parameters that has invalid values.
public function function getMissingParameters()
Returns an array that contains the names of missing required API parameters.
public function function invParams()
Sends a response message to indicate that a request parameter(s) have invalid values. This function will send back a JSON string in the following format:

{
  "message":"The following parameter(s) has invalid values: 'param_1', 'param_2', 'param_n'",
  "type":"error"
}

In addition to the message, The response will sent HTTP code 404 - Not Found.
public function function missingAPIAction()
Sends a response message to tell the front-end that the parameter 'action' is missing from request body. This function will send back a JSON string in the following format:

{
  "message":"Action is not set.",
  "type":"error"
}

In addition to the message, The response will sent HTTP code 404 - Not Found.
public function function missingParam(string $paramName)
Sends a response message to indicate that a request parameter is missing. This function will send back a JSON string in the following format:

{
  "message":"The parameter 'param_name' is missing.",
  "type":"error",
}

In addition to the message, The response will sent HTTP code 404 - Not Found.

Parameters:

  • string $paramName The name of the parameter.
public function function missingParams()
Sends a response message to indicate that a request parameter or parameters are missing. This function will send back a JSON string in the following format:

{
  "message":"The following required parameter(s) where missing from the request body: 'param_1', 'param_2', 'param_n'",
  "type":"error",
}

In addition to the message, The response will sent HTTP code 404 - Not Found.
public function function notAuth()
Sends a response message to indicate that a user is not authorized to do an API call. This function will send back a JSON string in the following format:

{
  "message":"Not authorized",
  "type":"error"
}

In addition to the message, The response will sent HTTP code 401 - Not Authorized.
public function function requestMethodNotAllowed()
Sends a response message to indicate that request method is not supported. This function will send back a JSON string in the following format:

{
  "message":"Method Not Allowed.",
  "type":"error",
}

In addition to the message, The response will sent HTTP code 405 - Method Not Allowed.
public function function send(string $conentType, type $data)
Sends Back a data using specific content type using HTTP code 200 - Ok.

Parameters:

  • string $conentType Response content type (such as 'application/json')
  • type $data Any data to send back (it can be a file, a string ...).
public function function sendResponse(string $message, boolean $isErr, int $code, string $otherJsonStr)
Sends a JSON response to the client. The basic format of the message will be as follows:

{
  "message":"Action is not set.",
  "type":"error"
} If other JSON string is given, It will be included in the response.

Parameters:

  • string $message The message to send back.
  • boolean $isErr [Optional] TRUE if the message represents an error state.
  • int $code [Optional] Response code (such as 404 or 200).
  • string $otherJsonStr [Optional] Any other data to send back (it should be a JSON string).
public function function setDescription(sting $desc)
Sets the description of the API.

Parameters:

  • sting $desc Action description. Used to help front-end to identify the use of the API.
public function function toJSON()
Returns JsonX object that represents the API.
