Class Functions Details

public static function function &get()
Returns a single instance of the class.
public function function __construct()
Creates new instance of the class. It is not recommended to use this function. Instead, use SystemFunctions::get().
public function function configured(boolean $isConfig)
Updates system configuration status. This function is useful when the developer would like to create some kind of a setup wizard for his web application.

Parameters:

  • boolean $isConfig [Optional] TRUE to set system as configured. FALSE to make it not configured.
public function function createConfigFile()
Creates the file 'Config. php' if it does not exist.
public function function getConfigVars()
Returns an associative array that contains system configuration info. The array that will be returned will have the following information:
  • is-config: A string. 'TRUE' or 'FALSE'
  • release-date: The release date of WebFiori Framework.
  • version: Version number of WebFiori Framework.
  • version-type: Type of WebFiori Framework version.
  • config-file-version: Configuration file version number.
  • database-host: The address of database server host.
  • database-username: The user that will be used to access the database.
  • database-password: The password of database user.
  • database-name: The name of the database.
  • database-port: Database host server number.
public function function isSetupFinished()
Checks if the application setup is completed or not. Note that the function will throw an exception in case one of the 3 main configuration files is missing.
public function function updateDBAttributes(string $dbHost, string $dbUser, string $dbPass, string $dbName, int $dbPort)
Update database attributes.

Parameters:

  • string $dbHost The name of database host. It can be an IP address or a URL.
  • string $dbUser The username of database user.
  • string $dbPass The password of the database user.
  • string $dbName The name of database schema.
  • int $dbPort Port number of database server.
